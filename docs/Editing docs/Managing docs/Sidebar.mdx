# Sidebar

You can configure a sidebar to manage your documentation, which is useful to:

- Group multiple **related documents**
- **Display a sidebar** on each of those documents



To use sidebars on your Docuo site:

1. Define a file that contains a dictionary of [sidebar objects](./Manange%20your%20docs.mdx#sidebar-object).
2. Use the sidebar key in the file in `docSidebar` item type of navbar.

<CodeGroup>

```json docuo.config.json
{
  "themeConfig": {
    "navbar": {
      "items": [
        {
          "type": "docSidebar",
          "label": "Docs",
          "sidebarIds": ["mySidebar"],
          "position": "left"
        }
      ]
    }
  }
}
```

</CodeGroup>

<CodeGroup>

```json sidebars.json
{
  "mySidebar": [
    {
      "type": "category",
      "label": "Overview",
      "items": ["overview/introduction"]
    },
    {
      "type": "link",
      "label": "Learn more",
      "href": "https://example.com"
    }
  ]
}
```

</CodeGroup>

## Sidebar items

There are four types of sidebar items including `doc`, `category`, `link`, and `autogenerated`.

| Item          | Rendered Output                                     |
| :------------ | :-------------------------------------------------- |
| Doc           | Link to a doc page, associating it with the sidebar |
| Link          | Link to any internal or external page               |
| Category      | Create a dropdown of sidebar items                  |
| Autogenerated | Generate a sidebar slice automatically              |

### Doc: link to a doc

Use the `doc` type to link to a doc page and assign the doc to a sidebar:

<CodeGroup>

```ts type.ts
type SidebarItemDoc =
  // Normal syntax
  | {
      type: "doc";
      id: string;
      label: string; // Sidebar label text
    }

  // Shorthand syntax
  | string; // docId shortcut
```

</CodeGroup>

`id` represents the unique identifier of a doc page, which is a string generated from the path of the doc page relative to the docs directory. The string is uniformly processed into pure lowercase, and spaces are replaced with `-` 

Here's an example:

<CodeGroup>

```json sidebars.json
{
  "mySidebar": [
    // Normal syntax
    {
      "type": "doc",
      "id": "overview/basic-features", // document ID
      "label": "Basic features" // sidebar label
    },

    // Shorthand syntax
    "overview/introduction" // document ID
  ]
}
```

</CodeGroup>

If you use the doc shorthand or [autogenerated](./Manange%20your%20docs.mdx#autogenerated) sidebar, you would lose the ability to customize the sidebar label through item definition.

<Warning title="WARNING">
A `doc` item sets an implicit sidebar association. Don't assign the same doc to multiple sidebars.
</Warning>



### Link: link to any page

Use the `link` type to link to any page (internal or external) that is not a doc.

<CodeGroup>

```ts type.ts
type SidebarItemLink = {
  type: "link";
  label: string;
  href: string;
};
```

</CodeGroup>

Here's an example:

<CodeGroup>

```json sidebars.json
{
  "myLinksSidebar": [
    // External link
    {
      "type": "link",
      "label": "Facebook", // The link label
      "href": "https://facebook.com" // The external URL
    },

    // Internal link
    {
      "type": "link",
      "label": "Home", // The link label
      "href": "/" // The internal path
    }
  ]
}
```

</CodeGroup>

### Category: create a hierarchy

Use the `category` type to create a hierarchy of sidebar items.

<CodeGroup>

```ts type.ts
type SidebarItemCategory = {
  type: "category";
  label: string; // Sidebar label text.
  items: SidebarItem[]; // Array of sidebar items.
};
```

</CodeGroup>

Here's an example:

<CodeGroup>

```json sidebars.json
{
  "mySidebar": [
    {
      "type": "category",
      "label": "Editing docs",
      "items": [
        "editing-docs/docs-components",
        {
          "type": "category",
          "label": "Managing docs",
          "items": [
            "editing-docs/managing-docs/sidebar",
          ]
        }
      ]
    }
  ]
}
```

</CodeGroup>

<Tip title="TIP">
Use the [**shorthand syntax**](./Manange%20your%20docs.mdx#category-shorthand) when you don't need customizations:
</Tip>

<CodeGroup>

```json sidebars.json
{
  "mySidebar": {
    "Editing docs": [
      "editing-docs/docs-components",
      {
        "Managing docs": ["editing-docs/managing-docs/sidebar"]
      }
    ]
  }
}
```

</CodeGroup>

### Using shorthands

You can express typical sidebar items without much customization more concisely with **shorthand syntaxes**. There are two parts to this: [**doc shorthand**](./Manange%20your%20docs.mdx#doc-shorthand) and [**category shorthand**](./Manange%20your%20docs.mdx#category-shorthand).

#### Doc shorthand

An item with type `doc` can be simplified to a string to represent its ID:

<CodeGroup>

```json sidebars(Longhand)
{
  "mySidebar": [
    {
      "type": "doc",
      "id": "overview/introduction"
    }
  ]
}
```

```json sidebars(Shorthand)
{
  "mySidebar": ["overview/introduction"]
}
```

</CodeGroup>

So it's possible for you to simplify the example above to:

<CodeGroup>

```json sidebars.json
{
  "mySidebar": [
    {
      "type": "category",
      "label": "Overview",
      "items": ["overview/introduction"]
    },
    {
      "type": "link",
      "label": "Learn more",
      "href": "https://example.com"
    }
  ]
}
```

</CodeGroup>

#### Category shorthand

A category item can be represented by an object whose key is its label, and the value is an array of subitems.

<CodeGroup>

```json sidebars(Longhand)
{
  "mySidebar": [
    {
      "type": "category",
      "label": "Overview",
      "items": ["overview/introduction", "overview/basic-features"]
    }
  ]
}
```

```json sidebars(Shorthand)
{
  "mySidebar": [
    {
      "Overview": ["overview/introduction", "overview/basic-features"]
    }
  ]
}
```

</CodeGroup>

This allows you to simplify the example to:

<CodeGroup>

```json sidebars.json
{
  "mySidebar": [
    {
      "Overview": ["overview/introduction"]
    },
    {
      "type": "link",
      "label": "Learn more",
      "href": "https://example.com"
    }
  ]
}
```

</CodeGroup>

Each shorthand object after this transformation will contain exactly one entry. Now consider the further simplified example below:

<CodeGroup>

```json sidebars.json
{
  "mySidebar": [
    {
      "Overview": ["overview/introduction"],
      "Getting started": ["getting-started/quickstart", "getting-started/development"]
    },
    {
      "type": "link",
      "label": "Learn more",
      "href": "https://example.com"
    }
  ]
}
```

</CodeGroup>



###  Autogenerated

Docuo can create a sidebar automatically from your filesystem structure: each folder represents a sidebar category, and each file generates a doc link.

<CodeGroup>

```ts type.ts
type SidebarItemAutogenerated = {
  type: "autogenerated";
  dirName: string; // Source folder to generate the sidebar slice from (relative to docs)
};
```

</CodeGroup>

Docuo can generate a full sidebar from your docs folder:

<CodeGroup>

```json sidebars.json
{
  "myAutogeneratedSidebar": [
    {
      "type": "autogenerated",
      "dirName": "." // "." means the current docs folder
    }
  ]
}
```

</CodeGroup>

An `autogenerated` item is converted by Docuo to a **sidebar slice** (discussed in [category shorthand](./Manange%20your%20docs.mdx#category-shorthand)): a list of items of type `doc` or `category`. So you can splice **multiple `autogenerated` items** from multiple directories, interleaving them with regular sidebar items, in one sidebar level.



### Default sidebar

If the `sidebarPath` is unspecified, Docuo [automatically use a default sidebar](./Manange%20your%20docs.mdx#autogenerated) for you by using the filesystem structure of the `docs` folder:

<CodeGroup>

```json sidebars.json
{
  "mySidebar": [
    {
      "type": "autogenerated",
      "dirName": "." // generate sidebar from the docs folder
    }
  ]
}
```

</CodeGroup>

You can also define your sidebars explicitly.



## Sidebar object

A sidebar at its crux is a hierarchy of categories, doc links, and other hyperlinks.

<CodeGroup>

```ts type.ts
type Sidebar =
  // Normal syntax
  | SidebarItem[]
  // Shorthand syntax
  | { [categoryLabel: string]: SidebarItem[] };
```

</CodeGroup>

For example:

<CodeGroup>

```json sidebars.json
{
  "mySidebar": [
    {
      "type": "category",
      "label": "Getting Started",
      "items": [
        {
          "type": "doc",
          "id": "getting-started/quickstart"
        }
      ]
    },
    {
      "type": "category",
      "label": "Overview",
      "items": [
        {
          "type": "doc",
          "id": "overview/introduction"
        },
        {
          "type": "doc",
          "id": "overview/basic-features"
        }
      ]
    },
    {
      "type": "link",
      "label": "Learn more",
      "href": "https://example.com"
    }
  ]
}
```

</CodeGroup>

This is a sidebars file that exports one sidebar, called `mySidebar`. It has three top-level items: two categories and one external link. Within each category, there are a few doc links.



## Complex sidebars example

Here's an complete example with complex sidebar usage:

<CodeGroup>

```json sidebars.json
{
  "mySidebar": [
    {
      "Editing docs": [
        "editing-docs/docs-components",
        {
          "type": "autogenerated",
          "dirName": "Managing docs"
        }
      ]
    },
    {
      "type": "autogenerated",
      "dirName": "Overview"
    },
    {
      "type": "link",
      "label": "Learn more",
      "href": "https://example.com"
    }
  ]
}
```

</CodeGroup>
